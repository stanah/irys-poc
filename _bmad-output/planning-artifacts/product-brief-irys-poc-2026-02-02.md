# Product Brief: irys-poc

## Executive Summary

irys-pocは、Web3技術を活用した分散型動画プラットフォームのPoCである。クリエイターが初日から、仲介者の搾取なく、ファンとの直接的な経済関係を透明に築ける場を提供する。

現行の動画プラットフォーム（YouTube等）はクリエイターに対し高い手数料（30%）、収益化条件（登録者1,000人・視聴4,000時間）、アルゴリズムによる露出制御を課している。既存の分散型代替（DTube、Odysee等）はこの構造的問題に挑んだが、UXの壁と「分散型であること」自体を価値提案にしたことで大規模普及に至っていない。

irys-pocは「クリエイターとファンの関係を永続的に守る」設計思想を核に、作品も支援も絆も消えないプラットフォームを実現する。動画1本あたり約$0.10という低コスト構造で持続可能性を確保しつつ、Web3であることを意識させないUXを最優先とする。将来的にはAIがプラットフォーム上で寄与するAI経済圏への自然な拡張を見据える。

---

## Core Vision

### Problem Statement

動画クリエイターは、プラットフォームによる構造的な搾取に直面している。YouTube等の中央集権型プラットフォームは収益の最大30%を手数料として徴収し、収益化には厳格な条件を課し、アルゴリズムがコンテンツの発見可能性を支配している。小規模クリエイターは条件を満たせず収益化の機会すら得られない。クリエイターはコンテンツの所有権を持たず、プラットフォームの方針変更に翻弄される。

### Problem Impact

- 小規模クリエイターの収益化機会の喪失（収益化条件未達成で活動継続を断念）
- クリエイターの30%の収益がプラットフォーム手数料として消失
- コンテンツの所有権がプラットフォームに帰属し、アカウント停止でコンテンツごと消失するリスク
- アルゴリズム依存による不公平な発見機会の分配
- クリエイターとファンの関係がプラットフォームに人質として囲い込まれている状態

### Why Existing Solutions Fall Short

分散型動画プラットフォームの先行者（DTube、Odysee、DLive、Theta等）は以下の理由で大規模普及に至っていない:

1. **「分散型であること」自体を価値提案にした** — 一般ユーザーはインフラの分散化に関心がない
2. **トークノミクス先行、UX後回し** — ウォレット設定やガス代の概念が一般ユーザーの最大の障壁
3. **「Web3=胡散臭い」という認知バイアス** — 暗号資産詐欺やNFTバブルの記憶が信頼構築を困難にしている
4. **独自トークン発行の法的リスク** — LBRY社のSEC敗訴・閉鎖が示す通り、トークノミクス設計の法的ハードルは高い
5. **「YouTubeの劣化版」を作ってしまった** — 同じ土俵で戦い、コンテンツ量・UXで敗北

### Proposed Solution

「Web3ネイティブのクリエイター直接支援プラットフォーム」として、以下を実現する:

- **Day 1からの収益化** — 登録者数・再生時間の条件なし。1人のファンでも経済関係が成立
- **ファンの支援が100%クリエイターに届く** — オンチェーンで検証可能な収益分配。プラットフォームは維持費のみ
- **作品も、支援も、絆も消えない** — 永続ストレージにより、プラットフォームが消滅してもコンテンツと支援記録は残り続ける。ファンの「あなたが最初の支援者です」という証明は永久に刻まれる
- **Web3を意識させないシンプルな体験** — 技術的複雑さを抽象化し、一般ユーザーが違和感なく使える
- **AIによる価値創出の段階的導入** — 自動字幕・翻訳・推薦等、AIがクリエイターの活動を直接支援する機能を段階的に提供

### Key Differentiators

1. **クリエイターとファンの関係を永続的に守る設計思想** — 利益最大化ではなく持続可能性のみを設計原理とすることで、一貫した判断と信頼を獲得。趣味プロジェクトであるがゆえに、投資家やVCに依存するスタートアップには不可能な設計上の自由度を持つ
2. **ファンの支援が100%届き、永久に残る** — 動画1本あたり約$0.10の低コスト構造により手数料搾取を排除。支援トランザクションはオンチェーンに永続記録され、クリエイターとファンの関係性が「デジタル時代の手紙」として刻まれる
3. **コンテンツの永続的所有権** — Irys永続ストレージにより、クリエイターの作品はプラットフォームの存続に依存しない。プログラマブルなアクセス制御で、関係性に応じた体験の自動化も可能
4. **AI経済圏への拡張性** — 人間向けサービスからAI+人間の経済圏へのピボットが設計上自然にできるアーキテクチャ。まずはAIによるクリエイター支援機能から段階的に導入
5. **法的リスクの回避設計** — 独自トークン不発行、SC直接送金型（P2P）、段階的サステナビリティモデルにより、先行者の法的失敗パターンを設計で回避

## Target Users

### Primary Users

#### Persona 1: 個人クリエイター（アーリーアダプター）

- **プロフィール:** 20代後半、テック系コンテンツクリエイター。Web3技術に関心があり、新しいプラットフォームを積極的に試す。既存プラットフォームでのフォロワー数は少〜中規模。
- **現在の課題:** 既存プラットフォームの収益化条件を満たせない、または手数料の高さに不満を感じている。良いコンテンツを作っているが、アルゴリズムに埋もれている。新しい収益モデルを探している。
- **動機:** 「良いコンテンツを作れば正当に報われる仕組み」への渇望。新しいテクノロジーへの純粋な好奇心。利益最大化よりも、作品が正しく評価される場を求めている。
- **成功の瞬間:** 初めてファンから投げ銭を受け取り、100%自分に届いたのをオンチェーンで確認した時。「たった1人でも、自分の動画に価値を感じてくれた」という実感。

#### Persona 2: コラボレーター（チーム制作クリエイター）

- **プロフィール:** 20代半ば、インディーコンテンツを複数人で制作。監督、作画、音楽等の役割分担あり。Web3については「面白そう」程度の認知。
- **現在の課題:** チームの貢献をどう分配するかが曖昧。既存プラットフォームでは収益が一人の口座にしか入らず、手動分配が面倒。チーム間で金銭的な話をすることへの気まずさ。
- **動機:** 自動的な収益分配でお金の話をせずにチームが回る仕組み。制作に何ヶ月もかかる作品が永久に残ること。
- **成功の瞬間:** スマートコントラクトで収益分配を設定し、投げ銭が入るたびに自動でSplitされるのを確認した時。「お金の話をしなくてよくなった」。

### Secondary Users

#### Persona 3: アクティブファン（貢献者 + キュレーター）

- **プロフィール:** 20代前半、推しクリエイターの熱心なファン。消費するだけでなく、字幕、翻訳、推薦など何かしらの形で貢献したいと考えている。新しいサービスを試してSNSでシェアするのも好き。
- **現在の課題:** 推しへの貢献方法が「お金を投げる」以外に限られている（例: YouTubeのコミュニティ字幕機能は廃止済み）。投げ銭の30%がプラットフォームに取られることへの不満。自分が「最初に見つけた」「最初から支えた」という記録が残らない。
- **動機:** 推しを応援したい純粋な気持ちが第一。報酬はおまけ。ただし貢献や支援の記録が永久に残るなら、それは嬉しい。良いクリエイターを広めてエコシステム全体に貢献したい気持ちもある。
- **成功の瞬間:** 自分がつけた字幕で海外ファンが増え、クリエイターから感謝された時。貢献履歴がオンチェーンに刻まれ、「初期サポーター」として可視化される。

### Future Scope Users

#### ディレクター / マネージャー（B2B）

- 小規模クリエイター事務所の代表・マネージャー層。複数クリエイターの収益管理、権限管理、ダッシュボード等のB2B機能が必要。PoCスコープ外とし、プラットフォーム成熟後に対応を検討。

### User Journey

#### クリエイターのジャーニー

| フェーズ | 体験 | 感情 |
|---------|------|------|
| **発見** | テック系コミュニティ/SNSで「手数料ゼロ、Day 1から収益化」の話題を見かける | 好奇心、半信半疑 |
| **オンボーディング** | ウォレット接続（MetaMask既所持）or アカウントアブストラクション（メール/SNSログイン） → プロフィール設定 → 最初の動画アップロード | 「意外と簡単」 |
| **初回価値体験（Aha!）** | 最初の投げ銭を受け取り、100%自分に届いたことをオンチェーンで確認 | 感動、「これが欲しかった」 |
| **習慣化** | 定期的に動画投稿。ファンとの関係性がオンチェーンで蓄積。既存プラットフォームとの並行運用 | 信頼、帰属感 |
| **拡張** | Revenue Splitでコラボ制作。AI字幕で海外ファンにリーチ拡大 | 成長実感 |

#### ファンのジャーニー

| フェーズ | 体験 | 感情 |
|---------|------|------|
| **発見** | 推しクリエイターが「新しいプラットフォームでも配信始めた」と告知。またはテック系SNSで話題を見かける | 推しへの付随 or 好奇心 |
| **オンボーディング** | アカウントアブストラクション（メール/SNSログイン）で即開始。ウォレットの存在を意識しない。Web3だと気づかないまま利用開始 | スムーズ、違和感なし |
| **初回価値体験（Aha!）** | 投げ銭が100%届くと知る。「最初の支援者」証明を獲得 | 嬉しさ、特別感 |
| **貢献** | 字幕投稿やコンテンツ推薦。貢献記録がオンチェーンに刻まれる | 誇り、推し活の充実 |
| **習慣化** | 支援・貢献履歴が蓄積し「古参ファン」としてのアイデンティティに。セルフカストディへの移行も選択可能 | 帰属感、成長 |

| **復号コスト実測** | 限定動画1再生あたりのLit Protocol復号コストを計測・記録 |

### Business Objectives

#### コスト構造の設計

| | 公開動画 | 限定動画 |
|---|---|---|
| **ストレージコスト（Irys）** | クリエイター負担（〜$0.10/本） | クリエイター負担（〜$0.10/本） |
| **暗号化** | なし | Lit Protocol |
| **復号コスト** | なし | 投げ銭/メンバーシップから補填 |
| **視聴者体験** | 完全無料 | 支援者/メンバー特典 |

#### 持続可能性の段階的達成

| フェーズ | 状態 | 運用コスト負担 | 達成条件 |
|---------|------|--------------|---------|
| **Phase 0（現在〜PoC）** | 技術実証 | 自己負担（最小限） | 公開/限定両パイプラインの動作確認 |
| **Phase 1（PoC後）** | クリエイター直接負担 | プラットフォーム運用のみ自己負担（目標: 月$20以下） | クリエイターがアップロード時にIrysストレージ費用を直接支払い |
| **Phase 2** | Tipによる補填 | ゼロに近づく | 限定動画の復号コストがTip/メンバーシップ収益で賄える。プラットフォーム維持費も極小手数料で補填 |

**核心的な成功基準:** stanah さん個人の運用コスト負担からの脱出。プラットフォーム固定コスト（ホスティング、ドメイン等）を月$20以下に抑え、Phase 1でクリエイターのストレージ費用直接負担を実現する。

#### 未決定事項・リスク

- **Lit Protocol Naga使用量課金の詳細が未公開** — 復号コストの実額がPoCで初めて判明する。1復号あたりのコストが高い場合、限定動画の視聴体験設計に影響
- **視聴者へのコストの見せ方** — メンバーシップに含まれるか、都度課金か、UX設計として将来決定

| **AA経由のオンボーディング** | メール/SNSログインで動画視聴・投げ銭が可能 |

| **復号コストの計測** | 限定動画1再生あたりのLitコストを記録 |
| **運用コストの把握** | プラットフォーム月間固定コストの実測 |
| **テスト動画** | 公開/限定合わせて10本以上のアップロード・再生確認 |
