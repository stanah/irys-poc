## Livepeer 詳細

### 概要

Livepeerは **分散型ビデオトランスコーディングネットワーク** である。動画配信において最もコストがかかるトランスコーディング（解像度・フォーマット変換）を、世界中のGPUオペレーターに分散させることで、従来の10分の1のコストを実現する。

### アーキテクチャ

```
┌─────────────────────────────────────────────────────────────────┐
│                    Livepeer Network Architecture                │
└─────────────────────────────────────────────────────────────────┘

  アプリケーション（配信者）
        │
        │ 動画ストリーム (RTMP/WebRTC)
        ▼
  ┌─────────────────┐
  │  Gateway Node   │  ← APIエンドポイント提供
  │  (Livepeer      │  ← ジョブのルーティング
  │   Studio等)     │  ← 課金管理
  └────────┬────────┘
           │
           │ ジョブ割り当て（LPTステーク量に比例）
           ▼
  ┌─────────────────────────────────────────────────────────────┐
  │                    Orchestrator Layer                       │
  │  ┌──────────┐  ┌──────────┐  ┌──────────┐                  │
  │  │Orchestr. │  │Orchestr. │  │Orchestr. │  ...             │
  │  │  Node A  │  │  Node B  │  │  Node C  │                  │
  │  └────┬─────┘  └────┬─────┘  └────┬─────┘                  │

  │       ▼             ▼             ▼                         │
  │  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
  │  │Transcod.│  │Transcod.│  │Transcod.│  GPU クラスター     │
  │  └─────────┘  └─────────┘  └─────────┘                     │
  └─────────────────────────────────────────────────────────────┘
           │
           │ トランスコード済み動画（HLS/DASH）
           ▼
       CDN / 配信

```

### 主要コンポーネント

#### 1. Orchestrator（オーケストレーター）
- ネットワーク上でトランスコーディングジョブを調整するノード
- LPTをステークして参加資格を得る
- ジョブ量はステーク量に比例して割り当てられる
- ETH手数料 + LPTインフレ報酬を獲得

#### 2. Transcoder（トランスコーダー）
- 実際にGPU/CPUでトランスコーディングを実行
- Orchestratorの配下で動作
- 複数解像度・フォーマットへの変換を担当

#### 3. Delegator（デリゲーター）
- LPTホルダーがOrchestratorにステークを委任
- 技術的な知識不要で参加可能
- 報酬の一部を受け取る

### P2P 動画配信の仕組み

```
┌─────────────────────────────────────────────────────────────────┐
│                 Theta P2P Video Delivery                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. コンテンツプロバイダーが動画をアップロード                 │
│                     │                                           │
│                     ▼                                           │
│  2. オリジンサーバーでHLSセグメント化                          │
│                     │                                           │
│                     ▼                                           │
│  3. 初期配信: エッジノード（シード）に配布                     │
│                     │                                           │
│      ┌──────────────┼──────────────┐                           │
│      ▼              ▼              ▼                           │
│   Edge A         Edge B         Edge C  (シードノード)         │
│      │              │              │                           │
│      ▼              ▼              ▼                           │
│  4. P2P伝播: 視聴者が近隣ピアからセグメント取得                │
│                                                                 │
│   👤←→👤←→👤       視聴者同士でセグメント交換                │
│   ↑↓   ↑↓   ↑↓      ・近隣ピア優先（低レイテンシ）            │
│   👤←→👤←→👤       ・足りない分はエッジノードから             │
│                                                                 │
│  5. 報酬: 中継量に応じてTFUELを獲得（Proof of Relay）          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Proof of Relay（中継証明）

エッジノードは実際に中継したデータ量を証明してTFUEL報酬を得る。

| 項目 | 詳細 |
|------|------|
| **報酬対象** | 動画セグメントの中継 |
| **証明方法** | ブロックチェーンにProof of Relayを提出 |
| **対象ノード** | Elite Edge Node（TFUELステーク必要） |
| **インセンティブ** | 中継量に比例したTFUEL報酬 |

3. **共通の学び**
   - 動画特化のブロックチェーン設計
